## 2024-05-24 - Firestore Synchronous Queries Blocking the Event Loop
**Learning:** The API Gateway uses `fastapi` and is structured asynchronously (`async def`). However, it performs sequential synchronous operations on the Google Cloud Firestore client (e.g., `db.collection("...").get()`). These synchronous queries block the main event loop, severely degrading the throughput of the entire FastAPI application while waiting for network I/O.
**Action:** Always offload synchronous Firestore queries inside asynchronous handlers to separate threads using `asyncio.to_thread`. If multiple independent synchronous queries are made in the same handler (e.g., aggregating stats from different collections), use `asyncio.gather` with `asyncio.to_thread` to execute them concurrently, drastically reducing overall latency.
