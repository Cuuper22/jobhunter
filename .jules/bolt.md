## 2024-05-24 - Firestore Synchronous Queries inside FastAPI Async Endpoints
**Learning:** The FastAPI setup (`api-gateway/main.py`) exposes `async def` endpoints, but uses a synchronous Firestore client (`db.collection().get()`). This causes synchronous network I/O to run directly on the main Uvicorn event loop, blocking all other requests and dramatically increasing tail latency under load. Furthermore, some endpoints performed multiple independent queries sequentially, causing an $O(N)$ latency penalty.
**Action:** Always wrap synchronous Firestore I/O calls in FastAPI endpoints with `await asyncio.to_thread(lambda: ...)`. If multiple independent queries are required (e.g., aggregation counts for dashboard stats), parallelize them using `await asyncio.gather(...)` to reduce total query latency from $O(N)$ to $O(1)$.
