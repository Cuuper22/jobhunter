## 2024-03-30 - Offloading and parallelizing synchronous Firestore queries in FastAPI
**Learning:** The Python Google Cloud Firestore client's `.get()` operations are synchronous and perform IO over the network. Running multiple `.get()` calls consecutively inside an `async def` FastAPI route blocks the asyncio event loop and leads to high endpoint latency. Since gRPC channels are thread-safe, it is safe to offload these calls using `asyncio.to_thread`.
**Action:** When working with the synchronous Firestore Python client in an async framework like FastAPI, offload synchronous database I/O using `asyncio.to_thread`. For multiple independent queries within the same endpoint, wrap the thread-offloaded calls in `asyncio.gather` to execute them concurrently in separate threads, significantly improving performance.
