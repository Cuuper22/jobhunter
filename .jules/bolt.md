## 2024-05-18 - Blocking the FastAPI Event Loop with Synchronous Firestore Queries
**Learning:** Using synchronous Firestore operations (like `.get()` or `.count().get()`) inside a standard `async def` FastAPI route blocks the underlying ASGI event loop. This leads to severe performance degradation under load because subsequent concurrent requests cannot be processed while the current thread is blocked on network I/O to Firestore. In endpoints that make multiple independent queries (like aggregating dashboard statistics), running them sequentially exacerbates the issue.
**Action:** When working with the synchronous Python Firestore SDK within an asynchronous web framework, wrap all I/O bound calls in `asyncio.to_thread()`. For endpoints that require multiple independent queries, offload each to a thread and execute them concurrently using `asyncio.gather()`. This pattern preserves the non-blocking nature of the event loop and significantly reduces latency.
